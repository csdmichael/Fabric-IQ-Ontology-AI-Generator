Recommended workflow
  1. Business user prompts the ontology generator in plain English.
  2. Business user reviews graph, entities, relationships, and properties, then saves draft.
  3. Business user submits draft to IT for Lakehouse bindings.
  4. IT configures Fabric connection, binds fields/views, and submits ontology package.
  5. Admin deploys packaged ttl/json artifacts to Fabric.
